Math Handwriting Recognition

  • Learning Platform Team: Notebooks Scrum

As a member of the Notebooks Scrum for the Chan Zuckerberg Initiative's Learning Platform team, I introduced handwriting recognition into the math editor of the Notebooks document editor. This allows students to convert their handwritten math into LaTeX that is then inserted into their document – enhancing the capability of students to be able to learn math units and complete homework in Notebooks. This feature has now been rolled out to select Summit schools participating in the pilot programming.

Logging UI Interactions

  • Learning Platform Team: Notebooks Scrum

As a member of the Notebooks Scrum for the Chan Zuckerberg Initiative's Learning Platform team, I integrated a Ruby on Rails logging system into the Notebooks (student document system) codebase to log student and teacher usage of editor functionality and determine areas for further improvement. This is a major improvement from the previous document editor on the Learning Platform because it allows us to draw conclusions about the way students employ tools to learn and work effectively in their document editor.

Code Haven Curriculum 2018-2019

  • Organization: Code Haven Yale
  • Role: Curriculum Director
  • Technologies: Scratch & MIT App Inventor

Through Code Haven, students will gain an introduction to computer science that covers the meaning of the discipline, the types of logical thought it involves, some key terms and concepts, and examples of computer science in the world. Through programming exercises and an extended project to develop a unique mobile app, our curriculum is designed to give students a foundation of knowledge to carry them through computer science in the future, but most importantly it focuses on attracting and developing an interest in CS to motivate all demographics of students to pursue CS in years to come.

Part 1 of the program focuses on key computer science concepts that will be taught in Scratch as well as through unplugged and hands-on robotics activities. Part 2 of the program will be an opportunity for students to build creative personal projects in MIT App Inventor that they will present at the Spring 2019 Code Haven Project Fair to the entire class of Code Haven students, teachers, and parents.

NLP is rapidly growing, and, as a result, advancing in the field can seem daunting to the student or the researcher. Using Multilingual Neural Re-ranking Models for Low Resource Target Languages in Cross-lingual Document Retrieval

Abstract: Low-resource target languages introduce many challenges for cross-lingual document retrieval and detection (CLDD), as well as re-ranking. First, while CLDD can be reduced to monolin- gual information retrieval by document translation using machine translation (MT) systems, such MT systems suffer from the lack of parallel data for low-resource target languages. Second, recent neural retrieval models that outperform traditional language modeling ap- proaches suffer from the scarcity of relevance judgments in low-resource target languages. Due to these constraints, it is necessary to find ways to optimize the document retrieval process in ways that are not bound by the restraints on training data. This project focuses on exploring a variety of existing monolingual neural re-ranking models and applying them to the task of multilingual document retrieval.

Publication: Zhang, R., Westerfield, C., Shim, S., Bingham, G., Fabbri, A., Verma, N., Hu, W., Radev, D. (2019) “Improving Low-Resource Cross-lingual Document Retrieval by Reranking with Deep Bilingual Representations.” Association for Computational Linguistics.

AT&T Customer Support Chat

  • Intern Coding Challenge Competition

The Intern Coding Challenge for Summer 2018 was to improve the customer support chat system for AT&T using data on customer satisfaction. As a team of seven, the Chicago team built a customer-facing interface, employee-facing interface, and a manager dashboard.

I focused primarily on the backend of the support chat system. Using Flask, AJAX requests, SQL queries, and a phpMyAdmin database hosted on Xampp, the backend allowed for new data analysis techniques. The main feature we added was sentiment analysis of the chat text using TextBlob since according to a study by Samsung researchers on their chat data found that chat sentiment was the highest predictor of customer satisfaction.

I also worked on the manager-facing dashboard which queried the database to retrieve live results on chat topics, number of ongoing chats, and average customer/employee sentiment. For this section of the project we used Chart.js and a Creative Tim Bootstrap template.
